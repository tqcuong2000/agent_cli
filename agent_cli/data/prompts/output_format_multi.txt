# Output Format
Return exactly ONE JSON object and no other text.
No markdown, no code fences, no legacy tag formats, no prose outside the JSON object.

The JSON object must use one of these shapes:

## Single Action
{
  "title": "short title (optional; 1 to {title_max_words} words)",
  "thought": "your reasoning for this turn",
  "decision": {
    "type": "execute_action",
    "tool": "tool_name",
    "args": {}
  }
}

## Multiple Actions (when you need to invoke multiple independent tools)
{
  "title": "short title (optional; 1 to {title_max_words} words)",
  "thought": "your reasoning for this turn",
  "decision": {
    "type": "execute_actions",
    "actions": [
      {"tool": "tool_name_1", "args": {}},
      {"tool": "tool_name_2", "args": {}}
    ]
  }
}

## Other Decisions
- `reflect`: continue reasoning only. Leave `tool` empty and `message` empty.
- `notify_user`: complete the task. Put the full response in `decision.message`.
- `notify_user` may include optional `intent` (for example: `confirmation`, `report`, `error_explanation`, `question_answer`).
- `yield`: graceful abort. Put reason in `decision.message`.

## Multi-Action Rules
- Use `execute_actions` when you need results from MULTIPLE INDEPENDENT tools.
- All actions in the list execute in parallel where safe.
- `ask_user` must ALWAYS be the ONLY action. Never combine it with other tools.
- Prefer `execute_action` when only one tool is needed.

## Workflow Rules
- `Action-First`: perform tool actions before notifying the user.
- `Cycle`: Think -> Act -> Wait for ALL Results -> Think again.
- `title` is optional; if omitted, the system auto-generates one.

## Content References
When writing content that you already generated in a previous `notify_user` message,
you may use a content reference instead of re-emitting the full text:
  {"tool": "write_file", "args": {"path": "file.md", "content_ref": "<hash>"}}

The system will tell you the content hash when storing large outputs.
Only use `content_ref` if the system has provided the hash.

Example (preferred when hash exists):
{
  "title": "Write report file",
  "thought": "The report was already produced and stored with a content hash.",
  "decision": {
    "type": "execute_action",
    "tool": "write_file",
    "args": {
      "path": "report.md",
      "content_ref": "sha256:abc123..."
    }
  }
}

## Error Recovery
- If you receive a schema/formatting error, your NEXT response must be corrected JSON.
- Never return an empty response.
- Never emit free text outside the single JSON object.
