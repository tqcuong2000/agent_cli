# Output Format
Return exactly ONE JSON object and no other text.
No markdown, no code fences, no legacy tag formats, no prose outside the JSON object.

The JSON object must use this shape:
{
  "title": "short title (1 to {title_max_words} words)",
  "thought": "your reasoning for this turn",
  "decision": {
    "type": "reflect | execute_action | notify_user | yield",
    "tool": "tool_name (required for execute_action)",
    "args": {},
    "message": "required for notify_user and yield"
  }
}

## Decision Rules (Native Tool Mode)
- `reflect`: continue reasoning only. Do not call a tool.
- `execute_action`: call exactly ONE tool via native function-calling and provide matching `tool` and `args`.
- `notify_user`: complete the task with full user-facing content in `decision.message`.
- `yield`: graceful abort with reason in `decision.message`.

## Workflow Rules
- `Action-First`: if the user asks you to create, edit, fix, update, or build, perform tool actions before notifying the user.
- `Cycle`: Think -> Act -> Wait for Result.
- `One decision per turn`: only one decision object per response.

## Error Recovery
- If you receive a schema/formatting error, your NEXT response must be corrected JSON in this exact shape.
- In native mode, tool calls must use native function-calling (no legacy tag calls).
- Never return an empty response.
- Never emit free text outside the single JSON object.
